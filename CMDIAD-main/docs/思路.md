# PIRN-CMPT 研究思路

## 一句话定位

本文方法是在 PIRN 的“正常原型重构”主线上加入结构化原型分配 SPA 和跨模态原型传输 CMPT，用于解决 RGB 与 3D/SN 多模态工业异常检测中的缺模态问题。

核心逻辑是：

> 训练时利用完整 RGB+SN 正常样本学习正常外观、正常几何和跨模态共享结构；测试时如果某一区域无法被正常原型和正常性通信模块有效重构，则该区域产生较高重构误差，被判定为异常。完整模态时走真实 RGB/SN 双分支；缺模态时才启用 CMPT 补齐缺失模态的伪原型和伪特征。

## 1. 问题设定

MVTec 3D-AD 数据层面提供 RGB 图像与 3D 数据。本文不直接使用原始点云作为主干输入，而是将 3D 数据转换为表面法向图，记为 SN 模态。因此：

- 数据层面：RGB + 3D。
- 模型输入层面：RGB + SN。

训练集只包含正常样本：

$$
\mathcal{D}_{train}=\{(x_i^{rgb},x_i^{sn})\}_{i=1}^{N}
$$

测试阶段允许模态缺失：

$$
(m^{rgb},m^{sn}) \in \{(1,1),(1,0),(0,1)\}
$$

其中：

- $(1,1)$ 表示 RGB 与 SN 都可用；
- $(1,0)$ 表示 SN 缺失，只保留 RGB；
- $(0,1)$ 表示 RGB 缺失，只保留 SN。

模型输出图像级异常分数 $s$ 和像素级异常热力图 $H$：

$$
(s,H)=f_\theta(x^{rgb},x^{sn},m^{rgb},m^{sn})
$$

## 2. 训练时数据流

训练阶段默认 RGB 和 SN 都可用。

1. RGB 图像进入冻结的 RGB Encoder，得到 patch token：

$$
E_{rgb}=F_{rgb}(x^{rgb})
$$

2. SN 图像进入冻结的 SN Encoder，得到 patch token：

$$
E_{sn}=F_{sn}(x^{sn})
$$

3. RGB/SN token 进入 APR 和 SPA，构建正常原型重构表示：

$$
Z_{rgb}^{spa},\quad Z_{sn}^{spa}
$$

4. MNC 在 RGB 与 SN 之间传递正常性信息，得到进一步重构特征：

$$
Z_{rgb}^{mnc},\quad Z_{sn}^{mnc}
$$

5. 通过编码特征与重构特征之间的误差生成异常图：

$$
H_{rgb}=d(E_{rgb},Z_{rgb}^{mnc}),\quad
H_{sn}=d(E_{sn},Z_{sn}^{mnc})
$$

6. 融合两个误差图，得到最终 heatmap：

$$
H=\lambda_{rgb}H_{rgb}+\lambda_{sn}H_{sn}
$$

图像级分数默认取 heatmap 的最大值：

$$
s=\max_{u,v}H(u,v)
$$

## 3. 完整模态测试流

完整模态测试时：

```text
RGB image -> RGB Encoder -> APR/SPA -> MNC -> RGB Error Map
SN image  -> SN Encoder  -> APR/SPA -> MNC -> SN Error Map
RGB Error Map + SN Error Map -> Fusion -> Heatmap
```

注意：完整模态测试时不使用 CMPT 伪分支参与评分。CMPT 可以在训练阶段学习跨模态映射，但 full mode 的最终检测只依赖真实 RGB 和真实 SN。

## 4. 缺模态测试流

### 4.1 SN 缺失

输入为：

```text
RGB image available
SN image missing
```

数据流为：

```text
RGB image -> RGB Encoder -> real RGB features
real RGB features/prototypes -> CMPT -> pseudo SN prototypes + pseudo SN features
real RGB branch -> RGB Error Map
pseudo SN branch -> pseudo SN Error Map
RGB Error Map + pseudo SN Error Map -> Fusion -> Heatmap
```

数学上：

$$
\hat{P}_{sn},\hat{E}_{sn}=CMPT(P_{rgb},E_{rgb})
$$

然后：

$$
H=\lambda_{rgb}H_{rgb}+\lambda_{cmpt}\hat{H}_{sn}
$$

### 4.2 RGB 缺失

流程对称：

```text
SN image -> SN Encoder -> real SN features
real SN features/prototypes -> CMPT -> pseudo RGB prototypes + pseudo RGB features
real SN branch -> SN Error Map
pseudo RGB branch -> pseudo RGB Error Map
SN Error Map + pseudo RGB Error Map -> Fusion -> Heatmap
```

## 5. 模块分工

### 5.1 APR

APR 的作用是维护和精炼正常原型，使原型能够覆盖训练样本中的正常局部模式。理论上 PIRN 中 APR 是带门控的原型更新机制，核心思想是只吸收可信正常上下文，避免异常或噪声污染原型。

当前工程中 APR 保留为轻量原型更新和可选增强项。实验显示强行增强 APR/MNC 未必稳定提升，因此主结果使用稳定 baseline。

### 5.2 SPA

SPA 替代原 PIRN 的 BPA。BPA 解决“原型坍缩”，SPA 在此基础上加入结构化角色划分：

- RGB-specific prototypes：学习颜色、纹理、表面外观；
- SN-specific prototypes：学习几何、法向、形变结构；
- Shared prototypes：学习 RGB 与 SN 共同支持的轮廓、形状、空间布局。

候选原型矩阵为：

$$
P_m=[P_m^{spec};P^{sh}]
$$

其中 $m\in\{rgb,sn\}$。

### 5.3 CMPT

CMPT 是缺模态补偿模块。它不是生成缺失图像，也不是补点云，而是在原型和特征层面补齐缺失模态：

$$
P_s^{spec},E_s \rightarrow CMPT \rightarrow \hat{P}_t^{spec},\hat{E}_t
$$

其中 $s$ 是可用源模态，$t$ 是缺失目标模态。

### 5.4 MNC

MNC 用于多阶段正常性通信。完整模态下，它在 RGB 与 SN 真实分支之间通信；缺模态下，它在真实分支与 CMPT 生成的伪分支之间通信。

当前工程中有三类 MNC 设置：

- baseline MNC：稳定的原型重构与门控跨模态重构；
- strong MNC：参数化较弱的 PIRN 近似，实验中定位下降；
- learnable MNC：可学习 gated cross-attention + decoder，定位更好，但图像级 AUROC 可能下降。

## 6. 当前实验结论

### 6.1 Rope 10-shot

| Setting | Image AUROC | Pixel AUROC | AU-PRO | AU-PRO-0.01 |
|---|---:|---:|---:|---:|
| Full baseline | 0.976 | 0.985 | 0.945 | 0.403 |
| Full learnable MNC | 0.944 | 0.991 | 0.940 | 0.411 |
| Missing SN, no learnable MNC | 0.971 | 0.954 | 0.900 | 0.359 |
| Missing SN, learnable MNC | 0.913 | 0.984 | 0.941 | 0.387 |
| Missing RGB, no learnable MNC | 0.964 | 0.936 | 0.790 | 0.298 |
| Missing RGB, learnable MNC | 0.957 | 0.979 | 0.858 | 0.351 |

结论：

- full mode 主结果应使用 baseline；
- CMPT 在缺 SN 时有效，能保持较高 image-level 检测能力；
- learnable MNC 主要提升定位，不稳定提升 image-level 排序；
- missing RGB 比 missing SN 更难，说明 SN -> pseudo RGB 映射弱于 RGB -> pseudo SN。

### 6.2 Tire 10-shot

| Setting | Image AUROC | Pixel AUROC | AU-PRO |
|---|---:|---:|---:|
| Full baseline | 0.702 | 0.807 | 0.530 |
| RGB-only | 0.710 | 0.861 | 0.643 |
| SN-only | 0.571 | 0.569 | 0.157 |
| 50-shot full | 0.743 | 0.858 | 0.616 |

结论：

- tire 中 SN 分支明显不可靠；
- full fusion 不一定优于 RGB-only；
- 后续论文中应说明不同类别的模态贡献存在差异。

## 7. 当前技术取舍

1. 完整模态主线：用稳定 baseline。
2. 缺模态主线：用 CMPT 补伪原型和伪特征。
3. Learnable MNC：作为定位增强消融，不作为 full 主结果默认模块。
4. 不采用 `grid16 + K10` 作为当前工程主设置，因为在简化实现下明显欠拟合。
5. 若后续追求更接近 PIRN，应实现完整 GRU APR、prototype-level GAT MNC 和 layer-wise decoder。

## 8. 论文叙述重点

论文中不应声称“完整复现 PIRN”。更准确的表述是：

> 本文以 PIRN 的原型重构思想为基础，保留正常样本原型重构和跨模态正常性通信的主线，进一步提出面向缺模态推理的结构化原型分配 SPA 与跨模态原型传输 CMPT。完整模态下模型使用真实 RGB/SN 分支完成重构检测；缺模态下通过 CMPT 在原型与特征层面补齐缺失分支，从而恢复少模态条件下的异常检测能力。
